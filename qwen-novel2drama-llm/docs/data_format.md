# 数据格式说明

训练数据为 JSONL，每行一个样本。字段必须是：

- `instruction`：任务指令，例如“请把下面小说片段改编成 60 秒竖屏分镜脚本”。
- `input`：小说片段、人物设定、剧情摘要或场景描述。
- `output`：人工整理的高质量目标答案。

## 示例

```json
{"instruction":"请把下面小说片段改编成一集 60 秒竖屏 AI 短剧分镜脚本","input":"林晚站在订婚宴门口……","output":"标题：订婚宴上的背叛\n镜头1：……"}
```

## 高质量样本标准

1. 输出结构稳定：标题、核心冲突、镜头、台词、提示词等字段固定。
2. 强短剧感：前 3 秒钩子、每 15 秒小反转、结尾悬念。
3. 强视觉化：人物动作、服装、场景、光影和运镜可拍可生成。
4. 强情绪：背叛、压抑、爆发、反杀、掉马。
5. 可复用：角色一致性关键词和场景关键词可以跨集使用。

## 数据来源限制

不要使用 GPT、Claude、Gemini 等闭源商业模型的输出作为训练数据。推荐使用自己整理的数据，或未来使用许可明确的开源模型生成数据后再人工审核。

## 标注建议

先人工标注 100 条高质量样本，训练 1.5B LoRA 验证方向；再扩充到 500-2000 条，覆盖大纲、分镜、视频提示词和配音台词。
