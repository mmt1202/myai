# P1 Test runner

The test runner executes the `tests_to_run` commands generated by a patch plan and writes a reproducible JSON report.

It is intentionally restricted. By default it only allows test commands:

- `python -m unittest ...`
- `python -m pytest ...`
- `pytest ...`

Other commands are skipped.

## Prerequisites

Create a patch plan first:

```bash
python scripts/build_context_index.py
python scripts/build_code_symbols.py
python scripts/create_patch_plan.py --task "add /models endpoint to api_server"
```

## Dry run

Dry run validates what would be executed without running tests:

```bash
python scripts/run_test_plan.py --plan outputs/patch_plan.json --dry-run
```

## Run tests

```bash
python scripts/run_test_plan.py --plan outputs/patch_plan.json
```

Custom output:

```bash
python scripts/run_test_plan.py \
  --plan outputs/patch_plans/api_health_model_version.json \
  --output outputs/test_runs/api_health_model_version.json
```

## Output fields

The report records:

- task
- created_at
- overall_status
- summary
- each command
- allow/skip status
- return code
- stdout
- stderr

## Agent workflow

Recommended flow:

```bash
python scripts/create_patch_plan.py --task "your change request"
python scripts/run_test_plan.py --plan outputs/patch_plan.json --dry-run
# edit files only after reviewing the plan
python scripts/run_test_plan.py --plan outputs/patch_plan.json
```

The runner does not edit files. It only validates tests from a plan.
